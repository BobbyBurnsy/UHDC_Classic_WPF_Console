IT Pro Summary

This is a localized graphical overlay for Microsoft Intune and Entra ID (Azure AD) administration. It leverages the Microsoft Graph API to provide Tier 1 technicians with a highly restrictive, contextual GUI for performing common device and identity actions. Operating strictly on Delegated Permissions through the technician's active Entra login session, the tool guarantees that it cannot grant technicians any access, rights, or capabilities they do not already natively possess within the tenant. It is designed specifically to accelerate live phone support by eliminating the need to navigate slow, complex Microsoft Endpoint Manager web portals while a user is waiting on hold.

When to Use This Tool

The Scenario

You have a frustrated user on the phone. They are staring at a blue BitLocker recovery screen on their laptop, or perhaps an executive just got a new iPhone and can no longer approve their Microsoft Authenticator pushes to log into their email. Traditionally, you would have to put the user on hold, log into the Azure portal, authenticate, and dig through multiple layers of menus to find their specific device or security profile.

The Fix

The technician clicks "Intune Menu" in the UHDC. The script instantly pops open a clean GUI populated with the user's devices. While still talking to the user, the technician clicks the laptop and hits "Get BitLocker Key," instantly reading the 48-digit code over the phone. Or, they type the executive's new cell phone number into the box and click "Add SMS," allowing the executive to immediately receive a text message to log in. The tool empowers the technician to resolve cloud-identity issues in seconds, keeping the user engaged and drastically reducing call times.

Interactive Training Engine

When a junior technician runs this tool with Training Mode enabled, the UHDC pauses execution at each critical phase. It displays a pop-up explaining why the action is necessary, what the in-person equivalent would be, and reveals the exact PowerShell code being executed under the hood. Here is a sample of what the trainee sees:

[TRAINING] STEP 1: QUERY INTUNE & ENFORCE DOMAIN BOUNDARIES

WHEN TO USE THIS:
This happens automatically when the Intune Menu opens. It searches Microsoft Entra ID for the target user or computer.

WHAT IT DOES:
We are querying the Microsoft Graph API. Crucially, we extract your technician email domain (e.g., @dfw.state.gov) and compare it against the target's UserPrincipalName. If you try to look up a user or device belonging to a different agency, the script will block the query to enforce tenant security boundaries.

IN-PERSON EQUIVALENT:
Logging into endpoint.microsoft.com, searching for the user, and verifying their department/agency matches your support scope before making changes.

$users = Get-MgUser -Filter "userPrincipalName eq '$TargetUser'"
if ($users.UserPrincipalName -notmatch $TechDomain) { throw 'Cross-Agency Block' }
[TRAINING] STEP 2: RETRIEVE BITLOCKER KEY

WHEN TO USE THIS:
Use this when a user is locked out of their laptop by a blue BitLocker recovery screen (usually caused by a BIOS update, docking station change, or multiple failed PIN attempts).

WHAT IT DOES:
We are querying the Microsoft Graph API for the 'BitlockerRecoveryKey' object associated with this specific Azure AD Device ID. This allows you to read the key to the user over the phone instantly.

IN-PERSON EQUIVALENT:
Logging into portal.azure.com, navigating to Microsoft Entra ID > Devices, searching for the computer name, and clicking 'Recovery Keys' to view the 48-digit code.

Get-MgInformationProtectionBitlockerRecoveryKey -Filter "deviceId eq '$($dev.AzureAdDeviceId)'"
[TRAINING] STEP 3: RETRIEVE CLOUD LAPS

WHEN TO USE THIS:
Use this when you need local administrator rights on an Entra-joined (Azure AD) machine to install software or bypass UAC, and the standard on-prem LAPS password doesn't work.

WHAT IT DOES:
We are querying the Graph API for 'deviceLocalCredentials'. This retrieves the rotating, cloud-managed local administrator password for this specific device.

IN-PERSON EQUIVALENT:
Logging into the Intune Portal, selecting the device, and clicking 'Local admin password' in the monitor pane.

Invoke-MgGraphRequest -Method GET -Uri "https://graph.microsoft.com/v1.0/deviceLocalCredentials?$filter=deviceId eq '$($dev.AzureAdDeviceId)'"
[TRAINING] STEP 8: ADD SMS MFA

WHEN TO USE THIS:
Use this to quickly help a user set up their new cell phone for text-message MFA without requiring them to navigate the complex Microsoft security setup pages while you are on the phone with them.

WHAT IT DOES:
We are injecting a new 'mobile' PhoneAuthenticationMethod directly into the user's Entra ID profile.

IN-PERSON EQUIVALENT:
Logging into Entra ID > Users > Authentication Methods, clicking 'Add authentication method', selecting 'Phone number', and typing in the new cell number.

New-MgUserAuthenticationPhoneMethod -UserId $ResolvedUser.Id -PhoneType "mobile" -PhoneNumber $newPhone

Technical Execution Pipeline

  1. Dynamic Domain Filtration (LDAP): Extracts the User Principal Name (UPN) of the currently logged-in technician via a local Get-ADUser query (Port 389/LDAP). It splits the UPN to capture the domain suffix and concatenates it to the target user's SAMAccountName. This prevents cross-tenant data leakage by mathematically restricting the Azure search to the technician's specific organizational unit.
  2. Graph API Authentication (HTTPS): Uses the Microsoft.Graph PowerShell module to silently authenticate to Entra ID over Port 443 (HTTPS) leveraging the technician's active login session. It requests a strictly defined array of Delegated Permissions (scopes) required to execute the actions, ensuring all Graph calls are bound by the user's native RBAC limits.
  3. Dual-Vector Device Enumeration: Performs two parallel queries against Graph API. First, it queries Get-MgDeviceManagementManagedDevice using the computer hostname string. Second, it resolves the user's Entra ID Object GUID and performs a reverse-lookup to find all mobile devices explicitly assigned to that user. The arrays are merged, deduplicated, and passed to the WPF XAML interface.
  4. Context-Aware XAML UI: Renders a WPF GUI that dynamically hides or shows administrative buttons based on the target OS. (e.g., The "Remote Wipe" and "Remove Passcode" buttons only populate if the selected list item is evaluated as iOS/Android).
  5. REST API Dispatching: When a button is clicked, it dispatches the corresponding Graph API command back over Port 443 directly to Microsoft's cloud infrastructure.

Q&A

Q: Do technicians need to be global administrators in Entra ID to use this tool?
No. Because the script relies on the Connect-MgGraph module, it passes the technician's currently logged-in Windows credentials to Azure via pass-through authentication. Operating on Delegated Permissions, Microsoft Graph evaluates their specific Role-Based Access Control (RBAC) rights. If a Tier 1 tech doesn't have the Intune rights to remote wipe a phone, the button click will simply return an "Insufficient Permissions" error to the GUI.
Q: What network ports are required for this tool?
It uses Port 389 (LDAP) for the initial local domain extraction. All interactions with Microsoft Entra ID and Intune operate exclusively over Port 443 (HTTPS).
Q: Why does the script use `Invoke-MgGraphRequest` to get the LAPS password instead of a standard cmdlet?
The Microsoft Graph PowerShell SDK is massive, and some of the newer or more niche beta endpoints (like Cloud LAPS credential retrieval) aren't always fully wrapped in reliable, easy-to-use cmdlets depending on the module version installed. Constructing the raw URI and executing a direct GET call guarantees we hit the exact endpoint we need to extract the password payload.
Q: Does the "Force Sync" button push Group Policy Updates (gpupdate /force)?
No, it triggers an Intune MDM sync. This instructs the device to check in with the Microsoft Endpoint Manager cloud to download new configuration profiles, compliance policies, or software deployments. It is the cloud equivalent of a gpupdate, but for Intune specifically.
