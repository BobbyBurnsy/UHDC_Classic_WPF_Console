IT Pro Summary

The Theme Manager is a standalone GUI utility designed to hot-swap the color palette of the Unified Help Desk Console. Because IT tools should fit the vibe of your specific department, this script allows administrators to instantly apply pre-built color profiles to the massive XAML interface without having to manually edit hundreds of lines of code.

Technical Execution Pipeline

  1. Theme Dictionary: The script contains a hashtable of themes. Every theme is designed with 6 mathematically unique hex codes. For example, the "Mainframe" theme uses #050F05, #020802, and #010401 for its background layers to ensure no two variables share the exact same string.
  2. State Detection: When executed, the script reads the raw text of UHDC.ps1. It scans the file for the unique "Primary Accent" hex code to automatically determine which theme is currently active.
  3. Unique Hex Replacement: Once the current theme is identified, the script performs a strict .Replace() operation, swapping the 6 old hex codes with the 6 new hex codes from the selected theme. This avoids complex RegEx parsing and guarantees the XAML structure remains perfectly intact.
  4. Compiler Awareness: After saving the file, the script checks if a compiled UHDC.exe exists in the directory. If it does, it alerts the administrator that the executable must be recompiled to bake the new colors into the binary.

Q&A

Q: Will changing the theme break the UI layout?
No. The Theme Manager uses a highly targeted string replacement strategy. It only looks for and replaces the exact 6-character hex codes defined in the dictionary. It does not touch the XML tags, margins, or grid definitions of the UI.
Q: Can I add my own custom theme?
Yes! Simply open ThemeManager.ps1 in a text editor and add a new line to the $Themes dictionary. Just ensure that the 6 hex codes you choose are unique to your theme so the script can accurately detect and replace them in the future.
Q: I applied a theme, but my UHDC.exe still looks the same. Why?
When you compile a PowerShell script into an .exe using PS2EXE, the code (and its colors) are permanently baked into the executable file. The Theme Manager modifies the source UHDC.ps1 file. To see the changes in the executable, you must run your compiler script again to generate a fresh .exe from the newly themed .ps1 file.
